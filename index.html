Part 2: Deploying to Vercel (Step-by-Step) We will use the GitHub Integration method. This is the industry standard because it allows for "Continuous Deployment"—meaning if you change one line of code in GitHub, Vercel updates your site instantly. Step 1: Push your code to GitHub Log in to your GitHub account. Click the + icon in the top right and select New repository. Name it (e.g., my-vercel-portfolio), keep it Public, and click Create repository. Upload your file. You can do this via the command line, or simply click the link inside GitHub that says "uploading an existing file". Drag your index.html into the upload box and click Commit changes. Step 2: Connect Vercel to GitHub Go to Vercel.com and Sign Up / Log In (Use "Continue with GitHub" for the easiest experience). On your Vercel Dashboard, click the "Add New..." button and select Project. You will see a list of your GitHub repositories. Find my-vercel-portfolio and click Import. Step 3: Configure and Launch Vercel will show you a "Configure Project" screen. Framework Preset: Leave this as "Other" (since we are using plain HTML). Root Directory: Leave as ./. Click Deploy. Step 4: Success! Vercel will take about 10-20 seconds to build your site. Once the screen turns full of confetti, click the snapshot of your website. You will be given a live URL (e.g., my-vercel-portfolio.vercel.app) that you can share with anyone in the world. Alternative: The "Drag and Drop" Method (No GitHub) If you don't want to use GitHub right now and just want the file online in 30 seconds: Install the Vercel CLI (Command Line Interface) if you have Node.js installed: npm i -g vercel Open your terminal/command prompt in your folder (my-webpage). Type vercel and hit Enter. Follow the simple prompts (Say "Y" to everything). It will give you a live production URL immediately in the terminal.
